# RAG Search Engine — Гибридный поиск по документам

**Гибридный поисковый движок** для семантического и лексического поиска по документам. Объединяет векторный поиск (FAISS), BM25 и ранжирование с помощью Reciprocal Rank Fusion (RRF), чтобы выдавать наиболее релевантные фрагменты текста.

---

## Возможности

- **Семантический поиск (Dense)** – использует `sentence-transformers` (all-MiniLM-L6-v2) для поиска по смыслу, а не только по ключевым словам.
- **Лексический поиск (BM25)** – находит точные совпадения слов и терминов, что важно для специфических запросов.
- **Гибридный поиск (Hybrid)** – комбинирует результаты обоих методов с помощью **Reciprocal Rank Fusion (RRF)**, что даёт более точные результаты, чем каждый метод по отдельности.
- **Предварительно построенный индекс** – FAISS-индекс и чанки хранятся локально, поиск происходит мгновенно.
- **Веб-интерфейс** – на базе **Gradio** для быстрого тестирования и демонстрации.

---

## Как это работает

1. **Индексация**: документы разбиваются на небольшие фрагменты (чанки), для каждого строится векторное представление (эмбеддинг) и сохраняется FAISS-индекс. Также сохраняются тексты чанков и метаданные, а для BM25 – токенизированные версии чанков.
2. **Поиск**:
   - **Dense** – запрос преобразуется в вектор и ищет ближайшие соседей в FAISS.
   - **BM25** – запрос токенизируется и ранжирует чанки по частотности терминов.
   - **Hybrid** – результаты обоих поисков объединяются через RRF, финальные чанки сортируются по комбинированному рейтингу.
3. **Вывод**: возвращаются топ-N фрагментов с текстом, метаданными и скорами.
